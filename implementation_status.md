# Статус реалізації та наступний план

Оновлено: 17 липня 2026

## Уже готово

### Проєкт і публікація

- JavaScript/TypeScript-монорепозиторій на React + Vite.
- Workflow для GitHub Pages; для публікації лишається push у `main` і вибір **GitHub Actions** у налаштуваннях Pages.
- Локальний запуск, тести, lint і production build налаштовані.

### Контент

- У каталозі є всі 77 документованих юнітів Імперії.
- Для кожного є бойові формули, дії, еволюції, footprint і доступні пасиви.
- Дані перевіряються схемою Zod.

### Бойове ядро

- Детермінований RNG і автобій.
- Формули шкоди/лікування, вибір цілей, HP і смерть.
- Сітка 3×5, перевірка зайнятості та дворядні лицарі 2×1.
- Frontline: melee діє лише по фронтовому ряду, ranged може цілити будь-кого.
- Mana, regeneration і cooldown.
- Статус-ефекти, тривалість і DoT.
- Guardian перенаправляє частину шкоди на сусіднього захисника.

### Game loop і Hub

- Старт забігу: життя, золото, складність, economic limit.
- Найм із перевіркою золота й лідерства.
- Вибір одного з трьох шляхів, реальний автобій проти тимчасових рейдерів, нагорода за перемогу й втрата життя за поразку.
- Окремі екземпляри юнітів з HP, досвідом і позицією.
- Лікування, воскресіння, еволюції та локальне збереження у браузері.
- UI для Hub, найму, сітки та запуску бою.

## Що ще потрібно зробити

### 1. Завершити бойові механіки

- Аури, відбиття шкоди, lifesteal і контратаки.
- Повні AoE-форми: ряд, стовпець, коло, ширина удару та сусіди цілі.
- Reposition, cleanse, purge, mana burn, resurrection і спеціальні умови дій.
- Реальне застосування resistance, armor penetration, critical hit і бонусів проти race type.
- Позиційні модифікатори, тригери поведінки та пріоритети дій.

### 2. Завершити Hub та економіку

- Economic limit як ресурс, пасивний дохід і витрати.
- Рудники, їх захоплення та постійний дохід.
- Нагороди шляхів: досвід, унікальні юніти, economic limit і рудники.
- Повноцінна ціна лікування/воскресіння та баланс лідерства.

### 3. Поліпшити інтерфейс бою

- Візуальний журнал раундів, HP-бари, mana, cooldown і статуси.
- Анімація ударів, лікування, смерті, Guardian та AoE.
- Екран результатів із нагородами, досвідом і змінами стану армії.
- Панель тактики: пріоритети цілі, пріоритети дій і IF–THEN тригери.

### 4. Баланс і QA

- Автоматичні масові симуляції боїв.
- Налаштування формул, вартостей, нагород і кривої складності.
- Browser end-to-end тест для циклу: найм → розміщення → шлях → бій → Hub.
- Перевірка мобільного та вузького браузерного layout.

### 5. Випуск веб-MVP

- Закомітити зміни й запушити їх у `main`.
- Увімкнути GitHub Pages через GitHub Actions.
- Перевірити опубліковану версію в браузері.

## Поточний пріоритет

Наступним реалізувати решту бойових механік із розділу 1, починаючи з AoE-форм, resistance/critical і реакційних пасивів. Після цього перейти до нагород та економіки Hub.
