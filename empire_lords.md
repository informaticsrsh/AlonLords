# Фракція Імперія — Лорди (за схемою LordType/Modifier, п.1-2)

Обидва лорди — `primary_faction: "empire"`, без `secondary_factions` (можна додати пізніше через спецнавик). Рівень лорда (`level`, 1–20, м'яка стеля на формулах) масштабує їхні унікальні пасивки через набір дрібних приростів за рівень, а не через один великий стрибок. Базові атрибути лорда (Лідерство/Тактика/Бойова сила тощо) тут навмисно не задані — залишено на окремий розгляд.

## Набір і відкриття юнітів

На старті забігу лорд може наймати лише базових юнітів своєї фракції, відкритих саме для нього. Спільний стартовий набір Імперії: **Латник**, **Ополченець-лучник** і **Провидець**. Генріх додатково починає з **Послушником**, а Артур — зі **Зброєносцем**.

Після кожних **трьох перемог на Небезпечному перевалі** лорд автоматично відкриває одного випадкового базового юніта, якого ще не має. Вибір фіксується seed'ом забігу, тому не змінюється під час перегляду екрана. Відкриття тривають, доки не буде доступний увесь поточний пул базових юнітів Імперії:

- Генріх відкриває Брата Тиші, Слідчого та Зброєносця.
- Артур відкриває Послушника, Брата Тиші та Слідчого.

Зараз безпосередньо наймаються тільки юніти 1-го тиру; сильніші тири, як і раніше, отримуються еволюцією. Коли всі базові юніти вже відкрито, лічильник складних перемог більше нічого не видає.

## Як читати блоки коду нижче (спільна логіка для обох лордів)

Кожен елемент `faction_bonuses` — це один `Modifier` зі схеми п.2. У нього завжди одні й ті самі поля, і нижче пояснено, що означає кожне з них — далі по документу ці пояснення не повторюються:

- **`id`** — унікальний технічний ідентифікатор конкретного модифікатора (для дебагу/посилань, на гру не впливає).
- **`trigger`** — подія з `BattleEvent` (п.2), яка "будить" цей модифікатор. Наприклад `ON_UNIT_KILL` означає "перевіряти цей модифікатор щоразу, коли будь-який юніт когось вбиває"; `ON_TURN_START` — "щосекунди/щохід"; `ON_DAMAGE_DEALT` — "щоразу, коли нанесено шкоду"; `"passive"` — модифікатор діє завжди, без прив'язки до конкретної миттєвої події (наприклад, впливає на те, як рахуються формули).
- **`condition`** — функція-фільтр: `true`/`false`. Якщо `false` — модифікатор цього разу просто не спрацьовує (подія відбулась, але лорда вона не стосується). Приймає `ctx` (контекст події — хто/кого/скільки, дивись нижче) і, де потрібно, сам об'єкт `lord`.
- **`effect`** — те, що фактично відбувається, якщо `condition` дав `true`. Приймає поточний стан бою (`state`), лорда (`lord`) і за потреби контекст події (`ctx`), і змінює якісь числа (шкоду, дух, швидкість тощо).
- **`priority`** — число, яке вирішує порядок застосування, якщо на одну подію реагує кілька модифікаторів одразу. Стандартні "базові" правила гри мають `priority: 0`; усе, що вище — застосовується з урахуванням/поверх базового; вище число не обов'язково "сильніше", воно просто визначає черговість виконання.
- **`duration`** — як довго діє ефект: `"battle"` (скидається після бою, напр. Бойовий дух), `"permanent"` (діє завжди, поки лорд у грі), `"stacks"` (накопичується, тут не використовується).

Також у коді часто зустрічається `ctx` — це "контекст події": об'єкт з даними саме цього конкретного випадку. Наприклад для `ON_UNIT_KILL` це буде `{killer: <юніт>, killed_unit: <юніт>}` — тобто хто саме вбив і кого саме вбили цього разу (див. `BattleEvent {type, source, target, payload}` з п.2, тут `killer`/`killed_unit` — це заповнені `source`/`target` для цієї конкретної події).

## Лорд 1: Генріх Щасливий

**Концепція**: чим вищий Бойовий дух армії (він же "Віра" — див. термінологію в п.1 game_concept.md, це один і той самий `battleSpirit`) — тим сильніші бонуси отримують юніти, і сам дух росте легше. Плата — усі бойові характеристики лорда передаються юнітам із заниженим коефіцієнтом (базово 70%), який пом'якшується з рівнем лорда, але ніколи не зникає повністю.

```js
{
  id: "empire_lord_henrik",
  name: "Генріх Щасливий",
  primary_faction: "empire",
  secondary_factions: [],

  faction_bonuses: [
    // --- 1. Полегшене накопичення Віри ---
    {
      id: "easy_faith",
      trigger: "ON_UNIT_KILL",
      condition: (ctx) => ctx.killer.faction === "empire", // ctx = контекст події вбивства; killer = юніт-вбивця; діє лише на вбивства юнітами Імперії
      effect: (state, lord) => {
        const mult = 1.3 + 0.03 * lord.level; // +3%/рівень, м'яка стеля 2.0
        state.battleSpirit += faith_points(ctx.killed_unit) * Math.min(mult, 2.0);
      },
      priority: 5,
      duration: "battle"
    },
    {
      id: "faith_resilience",
      trigger: "ON_UNIT_DEATH",
      condition: (ctx) => ctx.dead.faction === "empire",
      effect: (state, lord) => {
        const reduction = Math.max(0.5, 1 - 0.02 * lord.level); // втрата духу слабшає з рівнем, підлога 0.5
        state.battleSpirit -= loss_penalty * reduction;
      },
      priority: 5,
      duration: "battle"
    },
    // --- 2. Посилені пороги Бойового духу (переозначає стандартний Modifier з п.1) ---
    {
      id: "amplified_zeal",
      trigger: "ON_TURN_START",
      condition: (ctx) => ctx.unit.faction === "empire",
      effect: (state, lord) => {
        if (state.battleSpirit > 80) {
          state.unit.speed_modifier = 1.2 + 0.015 * lord.level;        // база 1.2 → до ~1.5 на 20 рівні
          state.unit.resist_bonus_all = 0.005 * lord.level;            // +0.5%/рівень усі resist, поки дух > 80
          state.unit.crit_bonus = 0.003 * lord.level;                  // +0.3%/рівень крит-шанс, поки дух > 80
        } else if (state.battleSpirit < 20) {
          state.unit.speed_modifier = Math.min(1.0, 0.8 + 0.01 * lord.level); // штраф слабшає, на 20 рівні зникає повністю
        } else {
          state.unit.speed_modifier = 1.0;
        }
      },
      priority: 10, // вищий за базовий priority=0 з п.1 → перекриває стандартні пороги 1.2/0.8
      duration: "battle"
    },

    // --- 3. Плата за силу Віри: занижена передача характеристик лорда юнітам ---
    {
      id: "diminished_presence",
      trigger: "passive",
      condition: (ctx) => ctx.unit.faction === "empire",
      effect: (state, lord) => {
        const power_factor = Math.min(0.70 + 0.015 * (lord.level - 1), 0.90); // 70% на 1 рівні → стеля 90% (не 100%)
        state.lord_stats_for_formulas = {
          battlePower: lord.battlePower * power_factor,
          vitality: lord.vitality * power_factor,
          crystalRegenSpeed: lord.regenSpeed * power_factor
        };
        // Усі hp_formula(lord)/Action.formula(lord, self) юнітів цієї армії
        // отримують саме ці занижені значення замість "сирих" характеристик лорда.
      },
      priority: 1, // застосовується до формул юніта РАНІШЕ за faction_bonuses з ефектами (щоб ті рахувались від вже заниженої бази)
      duration: "permanent"
    }
  ]
}
```

### Пояснення логіки (Генріх Щасливий)

**1. `easy_faith` — легше накопичення Віри від вбивств**
- Спрацьовує щоразу, коли будь-який юніт когось вбиває (`ON_UNIT_KILL`), але `condition` пропускає далі тільки якщо вбивця (`ctx.killer`) — юніт Імперії.
- `faith_points(ctx.killed_unit)` — це базова кількість Віри (Бойового духу) за вбитого ворога, як вона і так рахується стандартним правилом гри (п.1, `battleSpirit += faith_points(killed_unit)`) — Генріх це число не змінює.
- `mult` — множник, на який Генріх помножує цю базову кількість. Формула `1.3 + 0.03*level` означає: навіть на 1 рівні лорд дає +30% до отриманої Віри за кожне вбивство, і це росте ще на +3% за кожен наступний рівень, доки не впреться у стелю ×2.0 (`Math.min(mult, 2.0)`) — тобто максимум удвічі більше Віри з одного вбивства, ніж отримав би "звичайний" лорд.
- `state.battleSpirit += ...` — власне і є "легше накопичення": до загального показника Бойового духу армії додається не базове значення, а базове ×помножене.

**2. `faith_resilience` — втрата Віри при смерті свого юніта болить менше**
- Спрацьовує на `ON_UNIT_DEATH`, тобто коли гине будь-який юніт; `condition` фільтрує так, щоб рахувалась лише смерть юніта самої Імперії (`ctx.dead.faction === "empire"`) — смерть ворога тут ні до чого.
- `loss_penalty` — це стандартна "штрафна" величина, на яку й так падає Бойовий дух при втраті свого юніта (базове правило з п.1, `battleSpirit -= loss_penalty`); Генріх сам penalty не вигадує, він лише применшує його.
- `reduction` — коефіцієнт применшення: `1 - 0.02*level` означає, що на 1 рівні штраф зменшується лиш трохи (на 2%), а з кожним рівнем — ще на 2%, аж до підлоги `0.5` (тобто штраф ніколи не стане меншим за половину від стандартного — повністю "невразливим" до втрат Генріх не робить армію).
- Підсумок: `state.battleSpirit -= loss_penalty * reduction` — фактичний мінус духу за загиблого стає меншим, ніж був би без цієї пасивки.

**3. `amplified_zeal` — сильніші пороги Бойового духу (і пом'якшений штраф)**
- Це заміна стандартного правила з п.1 (там при духу > 80 — `speed_modifier = 1.2`, при духу < 20 — `speed_modifier = 0.8`). Тут `priority: 10` (вище за базовий `priority: 0`), тому саме ця версія логіки перекриває стандартну для Імперії Генріха.
- Спрацьовує щохід (`ON_TURN_START`) для кожного юніта Імперії окремо (`ctx.unit`), перевіряючи поточний `state.battleSpirit` арм ії.
- **Якщо дух > 80** (армія "натхненна"): 
  - `speed_modifier = 1.2 + 0.015*level` — базовий бонус швидкості дій (1.2) зростає ще на 1.5% за рівень, тобто на 20 рівні юніти діють помітно частіше, ніж стандартні 20% прискорення.
  - `resist_bonus_all = 0.005*level` — додатковий (понад усе інше) бонус до всіх resistances, що росте на 0.5% за рівень; це "купа маленьких бонусів", про яку йшлося в запиті — суто додатковий приємний ефект, якого в базовій грі немає взагалі.
  - `crit_bonus = 0.003*level` — так само додатковий крит-шанс, що з'являється тільки завдяки Генріху і росте на 0.3% за рівень.
- **Якщо дух < 20** (армія "занепала духом"): стандартний штраф `speed_modifier = 0.8` тут замінено на `Math.min(1.0, 0.8+0.01*level)` — тобто штраф з рівнем слабшає (юніт діє все менш рідко), і на 20 рівні `0.8+0.01*20 = 1.0` — штрафу не лишається взагалі.
- **Якщо дух між 20 і 80** — нічого особливого, `speed_modifier = 1.0` (як завжди).

**4. `diminished_presence` — плата за все це: занижена передача характеристик**
- `trigger: "passive"` означає, що це не реакція на конкретну подію, а постійне правило про те, як рахуються формули юнітів (воно "підмінює" вхідні дані ще до того, як `hp_formula(lord)`/`Action.formula(lord, self)` юніта взагалі порахує число).
- `power_factor` — це і є той самий "штраф 70%": формула `0.70 + 0.015*(level-1)` означає, що на 1 рівні коефіцієнт точно дорівнює 0.70 (70%, як і було в ТЗ), а далі росте на 1.5% за кожен наступний рівень — але `Math.min(..., 0.90)` не дає йому піднятись вище 90%. Тобто навіть на максимальному рівні Генріх ніколи не дає юнітам більше 90% від своїх реальних характеристик — це навмисно постійна "плата" за архетип, а не тимчасовий недолік початкового рівня.
- `state.lord_stats_for_formulas` — це і є підмінені (занижені) значення `battlePower`/`vitality`/`crystalRegenSpeed`, які реально підставляються в формули юнітів замість "сирих" характеристик лорда. Тобто коли будь-який юніт Імперії під командуванням Генріха рахує свою шкоду чи HP за формулою на кшталт `10+1*lord.battlePower` (п.6) — туди підставляється вже занижене число, а не оригінальне.
- `priority: 1` — застосовується раніше за інші модифікатори з ефектами (2 і 3 вище мають `priority: 5` і `10`), щоб ті рахувались вже від заниженої бази, а не навпаки.

### Прогресія по рівнях (орієнтир)

| Рівень | faith_gain_mult | loss_penalty_mult | speed_modifier (дух>80) | speed_modifier (дух<20) | power_factor (штраф передачі) |
|---|---|---|---|---|---|
| 1  | 1.33 | 0.98 | 1.215 | 0.81 | 0.70 |
| 5  | 1.45 | 0.90 | 1.275 | 0.85 | 0.76 |
| 10 | 1.60 | 0.80 | 1.35  | 0.90 | 0.835 |
| 15 | 1.75 | 0.70 | 1.425 | 0.95 | 0.90 (стеля) |

Ідея балансу: на низьких рівнях лорд дуже залежний від того, чи вдається втримати Бойовий дух високим (штраф 70% відчутний, а бонуси ще малі); до 20 рівня штраф пом'якшується лише до 90% (лишається постійна плата за архетип), а от негативний бік Бойового духу (дух<20) до 20 рівня зникає повністю — гравець, що грає на утримання духу, з часом майже не боїться "просідань".

## Лорд 2: Артур Милосердний

**Концепція**: коли здоровий юніт цього лорда б'є по вже підбитому ворогу — удар перетворюється на "Удар милосердя": наносить додаткову шкоду і, якщо після цього ворог лишається зовсім при смерті, добиває його. Пороги X (мін. HP% атакуючого), Y (макс. HP% цілі) і Z (поріг добивання) та сила бонусної шкоди — усе посилюється з рівнем.

```js
{
  id: "empire_lord_arthur",
  name: "Артур Милосердний",
  primary_faction: "empire",
  secondary_factions: [],

  faction_bonuses: [
    {
      id: "mercy_strike",
      trigger: "ON_DAMAGE_DEALT",
      condition: (ctx, lord) =>
        ctx.attacker.faction === "empire" &&
        ctx.attacker.hp_pct > mercyX(lord.level) &&
        ctx.target.hp_pct_before_hit < mercyY(lord.level),
      effect: (state, lord, ctx) => {
        const bonusPct = mercyBonus(lord.level);           // додаткова шкода, % від базової
        const bonusDamage = ctx.baseDamage * (bonusPct / 100);
        ctx.target.hp -= bonusDamage;

        const executeThreshold = mercyZ(lord.level) / 100;
        if (ctx.target.hp_pct_after < executeThreshold) {
          ctx.target.hp = 0; // "добитий Ударом милосердя"
        }
      },
      priority: 8,
      duration: "permanent"
    }
  ]
}

// Формули порогів (рівень 1-20, м'які стелі як і просив автор ТЗ)
function mercyX(level) { return Math.max(50, 90 - 2 * level); }   // 1: 88% → 20: 50% (підлога, завжди >50 крім кепу самого на межі)
function mercyY(level) { return Math.min(49, 18 + 1.6 * level); } // 1: 19.6% → 20: 49% (стеля <50)
function mercyZ(level) { return Math.min(19, 6 + 0.7 * level); }  // 1: 6.7% → 20: 19% (стеля <20)
function mercyBonus(level) { return 15 + 2 * level; }             // 1: 17% дод. шкоди → 20: 55%
```

### Пояснення логіки (Артур Милосердний)

**`mercy_strike` — крок за кроком**
- Спрацьовує на `ON_DAMAGE_DEALT` — тобто щоразу, коли будь-який юніт наносить будь-яку шкоду будь-кому (звичайна атака, навичка — байдуже).
- `condition` пропускає далі, тільки якщо виконані одночасно 3 умови:
  1. `ctx.attacker.faction === "empire"` — атакував саме юніт Артура.
  2. `ctx.attacker.hp_pct > mercyX(lord.level)` — сам атакуючий на момент удару достатньо здоровий (вище порогу X). Це і є умова "здоровий юніт" з ТЗ.
  3. `ctx.target.hp_pct_before_hit < mercyY(lord.level)` — ціль ДО цього удару вже була підбита нижче порогу Y (`hp_pct_before_hit` — HP цілі у відсотках саме перед нанесенням цього конкретного удару, не після).
- Якщо всі три `true` — вважається, що це не просто удар, а "Удар милосердя", і `effect` виконує два кроки:
  1. **Додаткова шкода**: `bonusDamage = ctx.baseDamage * (mercyBonus(level)/100)` — до вже нанесеної базової шкоди (`ctx.baseDamage`, тобто того, що юніт і так завдав би своєю звичайною атакою/навичкою) додається ще `mercyBonus(level)` відсотків цього ж значення, і ця сума додатково знімається з HP цілі.
  2. **Перевірка на добивання**: після того як шкода вже застосована, дивимось `ctx.target.hp_pct_after` (скільки % HP лишилось у цілі ПІСЛЯ обох частин удару). Якщо це значення нижче порогу `mercyZ(level)/100` — ціль вважається "добитою Ударом милосердя" і гине одразу (`ctx.target.hp = 0`), навіть якщо формально в неї ще лишалось трохи HP.
- `priority: 8` — досить високий пріоритет, щоб ефект застосовувався після того, як базова шкода вже порахована (бо формулі потрібне готове `ctx.baseDamage`), але це не критично конфліктує з іншими модифікаторами в цьому документі, бо вони належать іншому лорду.

**Формули порогів — чому саме так:**
- `mercyX(level) = max(50, 90 - 2*level)` — поріг "атакуючий має бути здоровим". На 1 рівні дуже суворий (88% — майже повне HP), і з кожним рівнем вимога слабшає на 2%, аж до підлоги 50% (нижче ТЗ і не дозволяло — "X завжди більше 50"). Тобто чим вищий рівень Артура — тим менш здоровим може бути власний юніт, і Удар милосердя все одно спрацює.
- `mercyY(level) = min(49, 18 + 1.6*level)` — поріг "ціль має бути підбита". На 1 рівні дуже вузький (~20% — ціль майже мертва), і з кожним рівнем розширюється на 1.6%, аж до стелі 49% (за ТЗ — "Y завжди менше 50"). Тобто з рівнем Артур починає "добивати милосердям" дедалі менш підбитих ворогів.
- `mercyZ(level) = min(19, 6 + 0.7*level)` — поріг миттєвого добивання. На 1 рівні майже непомітний (~7% — рідко спрацює), і з рівнем росте на 0.7%, аж до стелі 19% (за ТЗ — "Z завжди менше 20"). Що вищий рівень — то з більшим запасом HP ціль все одно "дораховується" як мертва.
- `mercyBonus(level) = 15 + 2*level` — сама додаткова шкода Удару милосердя, без стелі: на 1 рівні +17% до базової шкоди, на 20 рівні вже +55%.
- Спільна логіка всіх чотирьох формул: на низьких рівнях Артур — рідкісний "фінішер" (все має майже ідеально збігтися: майже здоровий проти майже мертвого), а на високих рівнях — практично гарантований інструмент зачистки підбитих ліній ворога, бо пороги X/Y/Z стають набагато м'якшими, а сама бонусна шкода — значно більшою.

### Прогресія по рівнях (орієнтир)

| Рівень | X (HP% атакуючого, поріг "здоровий") | Y (HP% цілі, поріг "підбитий") | Z (поріг добивання) | Бонус шкоди |
|---|---|---|---|---|
| 1  | 88% | ~20% | ~7%  | 17% |
| 5  | 80% | ~26% | ~9.5%  | 25% |
| 10 | 70% | 34% | 13%  | 35% |
| 15 | 60% | 42% | 16.5%  | 45% |
| 20 | 50% (підлога) | 49% (стеля) | 19% (стеля) | 55% |

Ідея балансу: на 1 рівні механіка спрацьовує рідко й обережно (треба майже повне HP атакуючого проти майже добитого ворога, невеликий добивний поріг) — типовий "фінішер" для випадкових ситуацій. До 20 рівня умови значно послаблюються (майже будь-який відносно здоровий юніт добиває майже будь-якого підбитого супротивника), а бонусна шкода зростає більш ніж утричі — лорд перетворюється на спеціаліста з "чистки" підбитих ворожих ліній.

## Наступні кроки
- Коли з'являться інші фракції або прямий найм вищих тирів, пул відкриття треба розширити відповідними фракційними юнітами.
- `loss_penalty` і `faith_points()` — функції зі стандартного `BattleSpiritState` (п.1 концепту), тут лише множаться модифікаторами лорда; самі базові формули не змінені.
- Якщо рівень лорда в грі не обмежений 20 — усі формули вище мають `Math.min/max`, тож коректно тримають стелю/підлогу і на вищих рівнях.
