# Технічний стек і план реалізації

## Ціль першої версії

Web-MVP з версією для ПК-оверлею: один нескінченний забіг, вибір лорда, Hub-фаза, розміщення юнітів на сітці, автобій, нагороди та зростання складності. До MVP входить реалізація всіх юнітів, описаних у затвердженій документації фракції. Без мережевих функцій і метапрогресії між забігами.

## Рекомендований стек

- **Мова і монорепозиторій:** JavaScript із TypeScript для типобезпеки, Node.js та `pnpm` workspaces. TypeScript компілюється у звичайний JavaScript, але захищає складні бойові схеми від помилок.
- **Вебгра:** React + Vite для екранів Hub, меню й налаштувань; Phaser 3 для 2D-поля бою, анімацій та введення. Гра працює в сучасному браузері без інсталяції.
- **Ігрове ядро:** окремий пакет `packages/game-core`, незалежний від React, Phaser і Electron. Він обробляє сітку, події бою, `Modifier`, формули, seedований RNG і стан забігу. Обидві версії запускають саме це ядро.
- **Дані контенту:** JSON-файли з перевіркою через Zod. Лорди, юніти, дії, фракції та модифікатори — дані, а не хардкод; помилковий контент виявляється під час збірки.
- **ПК-оверлей:** Electron — desktop-обгортка для тієї самої вебгри. Прозоре безрамкове вікно (`transparent`, `alwaysOnTop`) дає overlay поверх інших застосунків; за потреби можна ввімкнути/вимкнути перехоплення кліків. Окремий плагін не потрібен.
- **Збереження:** у браузері — `localStorage`/IndexedDB, у Electron — локальний файл через безпечний preload API. Формат сейву спільний.
- **Якість і збірки:**
  - **Vitest** — швидкі автоматичні тести правил у `game-core`: наприклад, чи правильно `Guardian` перехоплює шкоду, чи модифікатори застосовуються в потрібному порядку і чи той самий seed дає той самий результат бою.
  - **Playwright** — тестує готову вебгру в браузері як гравець: відкрити забіг, найняти й розставити юніта, почати бій та переконатися, що екран результату з'явився без помилок.
  - **ESLint** — знаходить потенційні помилки й небезпечні конструкції в JavaScript/TypeScript до запуску гри; **Prettier** автоматично приводить код до єдиного форматування.
  - **GitHub Actions** — запускає перевірки Vitest, Playwright, ESLint і збірку автоматично при кожному push або pull request, щоб зламаний код не потрапив у реліз.
  - **Vite** — перетворює вихідний код React/Phaser на оптимізовані HTML, JavaScript, CSS та ресурси для публікації вебверсії.
  - **Electron Builder** — пакує цю саму гру разом з Electron у Windows-застосунок (`.exe`-інсталятор), включно з налаштуваннями overlay-вікна.
- **Пізніше, за потреби:** PWA для встановлення вебверсії, аналітика та серверний профіль лише після перевірки core-loop.

## Короткий план робіт

1. **Каркас проєкту.** Створити JavaScript/TypeScript-монорепозиторій: `game-core`, web-клієнт і Electron-обгортка; налаштувати збірки та тести.
2. **Вертикальний зріз правил.** Визначити JSON-схеми, написати симулятор бою для 2–3 юнітів, сітку, вибір цілі, шкоду, смерть і seedований RNG.
3. **Модульна система бою.** Додати `BattleEvent`, `Modifier`, тригери, пріоритети та журнал бою; покрити ключові взаємодії Vitest-тестами.
4. **Перший ігровий цикл у вебі.** Реалізувати React-екрани Hub і розміщення, Phaser-поле бою, стан забігу, найм/лікування та вибір одного з трьох шляхів.
5. **Повний контент MVP.** Перенести в JSON і реалізувати 2 лордів та всі ~77 юнітів Імперії з `empire_units_full_schema_1.md`, включно з їхніми діями, пасивами й позиційними ефектами. Налаштувати винагороди та криву складності.
6. **ПК-оверлей і полірування.** Упакувати вебклієнт в Electron, додати режими overlay/звичайного вікна, локальні сейви, звук і Windows-збірку; провести балансування симуляціями.

## Критерій готовності MVP

Гравець може відкрити гру у браузері або в Windows-оверлеї, почати забіг, зібрати й розставити армію з усіх задокументованих юнітів Імперії, пройти кілька циклів Hub → вибір шляху → автобій, програти або завершити забіг без блокерів. Результат кожного бою відтворюваний за тим самим seed, а базові правила покриті тестами.
